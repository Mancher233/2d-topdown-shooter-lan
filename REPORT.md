# REPORT.md -- 课程报告

## 游戏名称：2D 俯视角射击游戏（局域网双人对战）

---

## 一、游戏操作说明

| 操作     | 按键           |
| -------- | -------------- |
| 移动     | WASD 或 方向键 |
| 瞄准     | 鼠标移动       |
| 射击     | 鼠标左键       |
| 扔手雷   | G 键           |
| 翻滚闪避 | 空格键         |

**游戏规则：**
- 两位玩家在一个有掩体的竞技场中对战
- 子弹会被墙壁阻挡，手雷可以飞过墙壁
- 翻滚时有无敌帧（约 0.4 秒），但翻滚后有 3 秒冷却
- 被击败后 3 秒自动重生
- 每颗子弹 12 点伤害，手雷爆炸 40 点伤害，满血 100

## 四、技术架构概述

### 整体架构：主机权威模式（Host-Authoritative）

```
┌──────────────┐     WebSocket      ┌──────────────┐
│   房主电脑    │ ←─────────────────→ │   服务器     │
│  (运行游戏    │                     │ (纯消息转发)  │
│   模拟+渲染)  │                     └──────┬───────┘
└──────────────┘                            │
                                    WebSocket│
                                            │
                                    ┌───────▼───────┐
                                    │   加入者电脑    │
                                    │  (发送输入     │
                                    │   + 渲染画面)  │
                                    └───────────────┘
```

**工作原理（通俗版）：**

1. **服务器** (`server/server.js`)：就像一个邮局。它不做任何游戏计算，只负责把 A 的信件转交给 B，把 B 的信件转交给 A。

2. **房主电脑**：运行完整的游戏逻辑。它同时控制两个玩家角色的移动、子弹飞行、手雷爆炸、伤害判定。大约每秒 30 次把完整的游戏状态"拍照"发给加入者。

3. **加入者电脑**：不运行任何游戏逻辑。它只负责两件事：(a) 把自己的按键和鼠标操作发给房主；(b) 接收房主发来的游戏"照片"，然后画到屏幕上。

### 关键技术点

- **HTML5 Canvas**：用于绘制所有游戏画面（玩家、子弹、地图、迷雾等）
- **WebSocket**：实现实时双向通信（比普通 HTTP 请求快得多）
- **碰撞检测**：使用简单的圆形-矩形碰撞（所有角色的 hitbox 都是圆形）
- **战争迷雾**：从玩家位置发射 72 条射线（每 5 度一条），碰到墙壁就停，然后把这些射线的终点连成一个可视多边形，在多边形外面覆盖黑色遮罩

## 五、局域网多人游戏的工作原理（通俗版）

想象你和朋友在同一个房间里，各自拿着一台电脑：

1. 你的电脑运行了一个"服务器程序"（`node server.js`），它监听 3000 端口
2. 朋友的电脑通过 Wi-Fi 连接到你的 IP 地址（比如 192.168.1.100:3000）
3. 连接建立后，两人的浏览器之间就建立了一条"实时管道"（WebSocket）
4. 当朋友按下 W 键，这个信息通过管道瞬间传到你的电脑，你的游戏程序就更新朋友角色的位置
5. 你的电脑把所有角色的位置"拍照"后通过管道发给朋友的电脑，朋友的屏幕就更新了

**为什么必须在同一个 Wi-Fi？**
因为局域网 IP 地址（如 192.168.x.x）只能在同一个网络内访问。不同网络（比如一个用手机流量、一个用 Wi-Fi）的设备无法直接通信。***需要特别注意的是，UCAS校园网疑似禁用了设备间联机。欲在校内联机，目前请使用手机热点等。***

## 六、文件结构说明

```
项目根目录/
  index.html        -- 游戏入口页面（大厅界面 + 游戏启动逻辑）
  css/
    style.css       -- 大厅界面的样式（游戏画面用 Canvas 绘制，不用 CSS）
  js/
    utils.js        -- 数学工具函数（距离、角度、碰撞检测等）
    input.js        -- 键盘和鼠标输入管理
    map.js          -- 地图定义（墙壁位置）和碰撞检测
    player.js       -- 玩家角色（移动、翻滚、瞄准、受伤、绘制）
    bullet.js       -- 子弹逻辑（飞行、撞墙、击中判定）
    grenade.js      -- 手雷逻辑（抛物线飞行、爆炸、范围伤害）
    vision.js       -- 战争迷雾（射线投射 + 视线遮挡）
    network.js      -- WebSocket 网络通信封装
    hostGame.js     -- 房主的游戏循环（模拟 + 渲染）
    clientGame.js   -- 加入者的游戏循环（只渲染）
  server/
    server.js       -- WebSocket 中继服务器
    package.json    -- Node.js 项目配置
  SETUP.md          -- 安装和运行指南
  REPORT.md         -- 本报告
```
