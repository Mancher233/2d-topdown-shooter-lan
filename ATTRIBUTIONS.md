# 声明与致谢

## 一、AI 辅助开发声明

在此对开发过程中使用的 AI 辅助工具进行说明。

### 1.1 AI 工具使用情况

本项目的主要开发工具为 **Qoder**（一款集成 AI Agent 的 IDE，模型为Qwen3.7-Max），在以下环节获得了 AI 辅助：

| 环节         | AI 参与程度 | 说明                                                                             |
| ------------ | ----------- | -------------------------------------------------------------------------------- |
| 架构设计     | 辅助        | 人工提出完整设想，AI 协助梳理模块划分和文件组织，最终方案由人工确认              |
| 核心游戏逻辑 | 辅助        | 移动、碰撞检测、子弹/手雷物理、战争迷雾射线等算法由人工提出思路，AI 协助编码实现 |
| 网络同步架构 | 辅助        | Host-Joiner 架构由人工设计，AI 协助实现 WebSocket 消息协议                       |
| 游戏机制     | 未使用      | 游戏的数值平衡和玩法惩罚机制完全由人工提出                                       |
| 角色与语音   | 辅助        | 功能需求由人工定义，AI 协助实现渲染、buff 系统、语音同步等                       |
| Bug 调试     | 辅助        | AI 协助分析运行时错误（如变量作用域问题），定位根因并提出修复方案                |
| Git 工作流   | 辅助        | 分支管理、PR 创建、squash-merge 等操作由人工发出指令， AI 执行                   |
| 美术资源     | 未使用      | 所有图片素材由人工提供，AI 未参与美术创作                                        |
| 语音资源     | 未使用      | 所有语音素材由人工提供，AI 未参与音频创作                                        |

### 1.2 AI 辅助的局限性说明

AI 在本项目中主要承担代码编写和调试辅助工作。以下关键决策完全由人工做出：
- 游戏类型和玩法设计
- 技术选型（纯 Canvas 2D + WebSocket，无框架，利用射线渲染War Fog）
- 数值平衡（射速、温度、伤害、冷却时间等）
- 角色和主题选择（张雪峰 vs 科比）
- 功能需求的优先级排序

AI 在开发过程中也产生了若干错误（如变量作用域 bug 导致游戏无法启动），这些错误均由人工发现并指导 AI 修复，最终上传至github issue进行归档整理。

## 三、技术引用与致谢

### 3.1 技术栈

本项目使用以下开源技术和标准：

- **HTML5 Canvas API** — 2D 游戏渲染
- **WebSocket** — 实时双向网络通信
- **Node.js** — 服务端运行时
- **Web Audio API** — 音频播放和反节流机制
- **原生 JavaScript (ES5)** — 无框架，模块化 IIFE 架构

本项目未使用任何第三方游戏引擎、框架或 npm 依赖。

### 3.2 算法与技术方案参考

| 技术点                | 说明                                                                      |
| --------------------- | ------------------------------------------------------------------------- |
| 战争迷雾 (Fog of War) | 基于射线投射 (Raycasting) 的视线系统，从玩家位置向四周发射射线，碰墙停止  |
| Host-Joiner 网络架构  | 房主运行完整游戏模拟，加入者仅负责渲染和发送输入                          |
| Delta-Time 驱动       | 所有游戏逻辑基于时间增量，确保不同刷新率显示器上行为一致（感谢群友提醒😘） |
| 反浏览器节流          | 利用 Web Audio API 播放音频                                               |
| 资源预加载与降级      | 图片/音频预加载系统，加载失败时自动回退到几何图形（渲染的问题搞了好久🥹）  |
